Detailed Description

Abstract base for 2D colliders attached to an Object.

A Collider defines a 2D collision shape for its owning Object and exposes a uniform API for point tests, pairwise collision tests, and debug drawing.

Integration & flow:

  • Ownership: a Collider is owned by an Object (see owner). Lifetime is tied to the Object's lifetime.
  • Broad phase: SpatialHashGrid partitions Objects and enumerates candidate pairs; ObjectManager then calls CheckCollision on those pairs.
  • Narrow phase: CheckCollision uses double dispatch via DispatchAgainst to invoke the correct shape–shape test.
  • Transform: if useTransformScale is true, SyncWithTransformScale is called from ObjectManager each frame to update internal scaled dimensions.
  • Debug: DrawDebug renders the shape using RenderManager in world space.

Notes:

  • All non-const behavioral APIs are virtual; concrete shapes (circle, AABB) must implement them.
  • Point coordinate spaces: point and rendering are in world space; shapes should incorporate the owner's transform and offset when testing.
