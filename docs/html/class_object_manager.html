Detailed Description

Owns, updates, draws, and queries scene objects.

ObjectManager stores objects as unique ownership, supports deferred addition via a pending list, keeps raw-pointer views for fast iteration, and exposes tag-based lookup. It also integrates a broad-phase spatial hash for collision checks and provides helper rendering entry points that batch draw calls through RenderManager.

Typical frame flow:

  • AddObject() queues new objects into pendingObjects.
  • UpdateAll() calls AddAllPendingObjects(), updates objects, and performs collision checks.
  • DrawAll() builds a render submission from rawPtrObjects (or a filtered subset) and flushes through RenderManager.
  • EraseDeadObjects() removes killed objects and updates indices/maps.

Adds a new object and returns a raw pointer to it.

Updates every active object and processes collisions.

1) Integrates pendingObjects into the active arrays. 2) Calls each object's Update(dt, engineContext). 3) Invokes CheckCollision() to run broad-phase queries and narrow-phase checks according to collisionGroupRegistry. 4) Cleans up killed objects with EraseDeadObjects().
