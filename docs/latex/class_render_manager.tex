\doxysubsection{Detailed Description}
Central rendering subsystem\+: resources, frame lifecycle, and draw submission. 

\doxylink{class_render_manager}{Render\+Manager} owns GPU resources (shaders, textures, meshes, materials, fonts, sprite sheets), manages render layers, prepares the offscreen scene target (FBO) when enabled, builds a batched render map, and flushes draw commands. It also exposes utilities such as viewport/scissor clearing, debug line drawing, and compute-\/dispatch support.

Typical per-\/frame flow\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{renderManager.BeginFrame(engineContext);}
\DoxyCodeLine{\textcolor{comment}{//\ ...\ game\ state's\ ObjectManager\ submits\ objects\ via\ StateManager\ -\/>\ RenderManager\ ...}}
\DoxyCodeLine{renderManager.EndFrame(engineContext);}
\DoxyCodeLine{\textcolor{comment}{//\ After\ EndFrame,\ a\ GameState\ may\ run\ PostProcessing()\ using\ the\ scene\ color\ texture.}}

\end{DoxyCode}
